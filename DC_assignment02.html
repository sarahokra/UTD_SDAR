1. Analyze the data
a. Dates
library(readxl)
library(dplyr)
library(stringr)
library(zoo)        # as.yearmon() 讓 "YYYY-MM" 轉成月份日期


# 1) 讀檔：跳過前兩列的說明列
raw <- read_excel("TrumpHarrisElection_download.xlsx",
                  sheet = "TrumpHarrisElection", skip = 2, col_names = FALSE)
names(raw) <- c("date","Trump","Harris","election")

# 2) 清理：僅保留像 2004-01 的列，處理 "<1"
df <- raw %>%
  mutate(date = str_trim(as.character(date))) %>%     # 去空白，確保是字串
  filter(str_detect(date, "^\\d{4}-\\d{2}$")) %>%     # 只留 YYYY-MM
  mutate(
    # 把 YYYY-MM 變成每月的第一天（Date 類型）
    date = as.Date(as.yearmon(date)),
    # 把 "<1" 變 0，再轉數值
    across(c(Trump, Harris, election), ~ as.numeric(str_replace(.x, "<1", "0")))
  ) %>%
  arrange(date)

# 3) 起訖時間
start_date <- min(df$date)
end_date   <- max(df$date)
start_date; end_date
[1] "2004-01-01"
[1] "2025-09-01"

The time range of the dataset is from 2004-01-01 to 2025-09-01.

b. Intervals
# 4) 時間間隔（天），確認是否為月度資料
gap_days <- diff(df$date)
summary(gap_days)
Time differences in days
   Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
  28.00   30.00   31.00   30.44   31.00   31.00 
# 也可用「月份」為單位檢查
gap_months <- diff(as.yearmon(df$date)) * 12
summary(gap_months)
   Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
      1       1       1       1       1       1

  • The data are collected on a monthly basis, with most intervals being 30 or 31 days.

  • The summary shows that the minimum interval is 28 days and the maximum is 31 days, which corresponds to the varying number of days in each month.

b. Use gtrendsR package to collect data (use gtrendsR01.R program)

Searching “Trump”,“Harris”, & “election” on Google Trends

# 這段程式碼會顯示，但不會執行

#install.packages("gtrendsR")

library(gtrendsR)

# ---- "Trump","Harris","election" 查詢 (不限時間地點) -----
TrumpHarrisElection = gtrends(c("Trump","Harris","election"), time = "all") # 在 Google Trends 查詢「Trump」 、「Harris」和「election」(全時間、地點)，資料存成TrumpHarrisElection
plot(TrumpHarrisElection)  #繪製TrumpHarrisElection裡儲存的資料

Searching “tariff” on Google Trends

# 這段程式碼會顯示，但不會執行

# ---- "tariff" 查詢 (不限時間地點) -----
plot(gtrends(c("tariff"), time = "all"))  # 繪製[Google Trends 查詢「tariff｣(全時間、地點)]的資料

# ---- "tariff" 查詢 (不限時間、特定地點) -----
data("countries") #載入Google Trends的國家代碼
plot(gtrends(c("tariff"), geo = "GB", time = "all")) #繪製[Google Trends 查詢英國「tariff｣(全時間)]的資料
plot(gtrends(c("tariff"), geo = c("US","GB","TW"), time = "all")) #繪製[Google Trends 查詢美國、英國、台灣「tariff｣(全時間)]的資料

# ---- "tariff","China military", "Taiwan" 查詢 (不限時間地點) -----
plot(gtrends(c("tariff","China military", "Taiwan"), time = "all")) # 繪製[Google Trends 查詢「tariff」 、「China military」和「Taiwan」(全時間、地點)]的資料

# ---- "tariff","China military", "Taiwan" 查詢-其他代碼 -----
tct = gtrends(c("tariff","China military", "Taiwan"), time = "all")  # 在Google Trends 查詢「tariff」 、「China military」和「Taiwan」(全時間、地點)]，資料存成tct
tct = data.frame(tct$interest_over_time) # 取出tct清單中的 interest_over_time 表格；轉成 data.frame 格式，並覆蓋原本的 tct
# ---- 補充代碼 -----
par(family="Georgia") #圖表設為Geogia字體

tg = gtrends("tariff", time = "all") # 在 Google Trends 查詢「tariff｣時間不限，資料存成tg
tg_iot = tg$interest_over_time # 把 Google Trends 的「時間序列搜尋熱度」單獨存到tg_iot

data("categories") #載入Google Trends的主題分類名稱(e.g., Health)

c. Save the data into csv and R formats.

Save the data into csv

TrumpHarrisElection_iot <- TrumpHarrisElection$interest_over_time

# 存成 CSV 檔
write.csv(TrumpHarrisElection_iot, "TrumpHarrisElection.csv", row.names = FALSE)

# 這段程式碼會顯示，但不會執行

# 存成 RData 格式

d. What are the differences between the two methods?

Download data manually Use ‘gtrendsR’ script
Time interval Select “Past Day”; data may be hourly or every 8 minutes. Select “All Time”, data is usually monthly (YYYY-MM). Freely specify time = “today 1-m”, “today 12-m”, “all”, or other date ranges.
Geography Manually switch to a single region (e.g., Global, US). Set geo directly in the code as “US”, “TW”, “GB”. Compare multiple countries at once.
Data structure The downloaded CSV file only has a few columns, such as “Time” and “Search Popularity for Each Keyword”.

Return a list containing multiple data frames:

• $interest_over_time (time series)

• $interest_by_region (search volume by region)

• $related_queries (related search queries)

• $related_topics (Related Topics)
Advantages Easy to use, ideal for quickly checking trends or making basic comparisons. Integrate processes (automated analysis, archiving, plotting) in R for easy reproducibility and sharing.
Limitations Requires manual operation and cannot be automatically updated or processed in batches. May encounter Google Trends API rate limits (429 error). If called too frequently, may be temporarily blocked.
