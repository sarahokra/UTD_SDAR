a. Download ten most recent documents from “Foreign Relations Committee” using govtdata01.R

## Scraping Government data
## Website: GovInfo (https://www.govinfo.gov/app/search/)
## Modified to download 10 most recent documents from Foreign Relations Committee

# Start with a clean plate and lean loading to save memory
gc(reset=T)

# install.packages(c("purrr", "magrittr", "readr"))
library(purrr)
library(magrittr)
library(readr)

## ============ RELATIVE PATHS (for RStudio Project) ============

# Path to your downloaded CSV file (put it in UTD_SDAR folder)
csv_file_path <- "govinfo-search-results-2025-12-04T00_41_54.csv"

# Directory to save the downloaded PDFs
# This will create a "govfiles" folder inside UTD_SDAR
save_dir <- "govfiles/"

# Create the folder if it doesn't exist
if (!dir.exists(save_dir)) {
  dir.create(save_dir)
  cat("Created folder:", save_dir, "\n")
}

# ============================================================

## Read CSV file (skip first 2 rows as header info)
govfiles <- read.csv(file = csv_file_path, skip = 0)

# Check the data
head(govfiles)
colnames(govfiles)

# Get PDF links and IDs
pdf_govfiles_url <- govfiles$pdfLink
pdf_govfiles_id <- govfiles$index

# Check how many files are available
cat("Total files available:", length(pdf_govfiles_url), "\n")

# Show first 10 PDF links
cat("\nFirst 10 PDF links to download:\n")
head(pdf_govfiles_url, 10)

# Function to download PDFs
download_govfiles_pdf <- function(url, id) {
  tryCatch({
    destfile <- paste0(save_dir, "govfiles_", id, ".pdf")
    download.file(url, destfile = destfile, mode = "wb")
    Sys.sleep(runif(1, 1, 3))  # Random sleep 1-3 seconds to avoid being blocked
    return(paste("Successfully downloaded:", url))
  },
  error = function(e) {
    return(paste("Failed to download:", url))
  })
}

# ============ DOWNLOAD 10 MOST RECENT DOCUMENTS ============

start.time <- Sys.time()
message("Starting downloads - 10 most recent documents from Foreign Relations Committee")

# Download only the first 10 documents
results <- 1:10 %>% 
  purrr::map_chr(~ download_govfiles_pdf(pdf_govfiles_url[.], pdf_govfiles_id[.]))

message("Finished downloads")
end.time <- Sys.time()
time.taken <- end.time - start.time

# Print results
cat("\n========== DOWNLOAD RESULTS ==========\n")
print(results)
cat("\nTime taken:", time.taken, "\n")
cat("Files saved to:", save_dir, "\n")

2. Write simple report on difficulties encountered in the scraping process.

a. How useable the scraped data?

The scraped data from GovInfo is relatively useable. The CSV file contains structured metadata including document titles, publication dates, PDF links, and collection types. However, there are some limitations:

  1. File naming: The downloaded PDFs are named generically (govfiles_1.pdf, govfiles_2.pdf), which makes it difficult to identify the content without opening each file.

  2. Mixed content types: The search results include various document types (Congressional Records, Bills, Reports), which may require additional filtering for specific research purposes.

  3. Large download size: The default export contains 1000 records, but we only needed 10 documents. There is no option to customize the number of records exported.

b. How to improve?

  1. Better file naming: Modify the R script to use meaningful filenames based on document titles or package IDs instead of generic numbers.

  2. Add filtering: Include code to filter documents by collection type, date range, or keywords before downloading.

  3. Error handling: Add more robust error handling to track which downloads succeeded or failed.
